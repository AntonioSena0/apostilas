1. Enunciado

Você foi contratado para gerenciar a segurança do banco de dados do Hospital Central. Sua missão é garantir que cada funcionário acesse apenas o necessário (Princípio do Privilégio Mínimo). Você deverá criar perfis de acesso distintos, testar as restrições e realizar uma auditoria para identificar possíveis brechas de segurança.

2. Preparação

  • Ferramenta: MySQL Workbench, DBeaver, PHPMyAdmin ou no terminal MySQL.
  • Usuário: Você deve estar logado como root para realizar as configurações.
  • Ambiente: Certifique-se de que o servidor MySQL está rodando localmente.

3. Script de Preparação (MySQL)

Execute o script abaixo para criar a estrutura do hospital e a massa de dados inicial:

CREATE DATABASE hospital_db;
USE hospital_db;

-- Tabela com dados públicos de agendamento
CREATE TABLE agendamentos (
    id_agenda INT AUTO_INCREMENT PRIMARY KEY,
    paciente_nome VARCHAR(100),
    data_consulta DATE,
    hora_consulta TIME,
    medico_responsavel VARCHAR(100)
);

-- Tabela com dados sensíveis de prontuário
CREATE TABLE prontuarios (
    id_prontuario INT AUTO_INCREMENT PRIMARY KEY,
    paciente_nome VARCHAR(100),
    diagnostico TEXT,
    medicacao_prescrita TEXT,
    historico_familiar TEXT
);

-- Inserindo dados de teste
INSERT INTO agendamentos (paciente_nome, data_consulta, hora_consulta, medico_responsavel) VALUES

4. Desafios de Prática

Parte 1: Gestão de Identidade

  1. Crie um usuário chamado 'recepcao_central'@'localhost' com a senha 'SenhaRec#123'.
  2. Crie um usuário chamado 'medico_geral'@'localhost' com a senha 'Med@Secure!2026'.

Parte 2: Concessão de Privilégios

  1. Dê ao usuário da recepção apenas a permissão de consultar (SELECT) e inserir (INSERT) na tabela de agendamentos. Ele não deve ver a tabela de prontuarios.
  2. Dê ao médico permissão total (ALL PRIVILEGES) em ambas as tabelas do banco hospital_db.

Parte 3: Teste de Restrições

  1. Abra uma nova conexão no seu SGBD com o usuário da recepção.
  2. Tente executar um SELECT * FROM prontuarios;. O que aconteceu?
  3. Tente inserir um novo agendamento. Funcionou?

Parte 4: Auditoria e Refinamento

  1. Como root, utilize uma query na tabela mysql.user para listar todos os usuários e verificar quais plugins de autenticação eles estão usando.
  2. Revogue o privilégio de DELETE do médico na tabela de prontuarios (ele pode ler e editar, mas nunca apagar um histórico médico).
  3. Verifique as permissões finais de ambos os usuários utilizando o comando SHOW GRANTS.

5. Orientações Finais

  • Evite o %: Nesta atividade, foque no localhost para garantir que o acesso seja estritamente local.
  • Logs de Erro: Preste atenção nas mensagens de erro do MySQL (como o Error 1142). Elas são suas melhores amigas para entender se o privilégio funcionou.
  • Entregável: Salve todos os comandos utilizados em um arquivo .sql e anote os resultados dos testes de acesso (o que foi bloqueado e o que foi permitido).
  • Use SHOW GRANTS FOR 'usuario'@'localhost'; frequentemente para validar suas configurações.
  • Teste cada restrição cuidadosamente — erros são oportunidades de aprendizado!
