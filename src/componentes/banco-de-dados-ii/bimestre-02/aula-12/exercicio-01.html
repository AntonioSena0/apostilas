Enunciado: Gestão de Vendas e Comissões

Uma empresa de varejo possui um banco de dados para monitorar o desempenho de sua equipe. O analista de dados precisa extrair relatórios específicos utilizando os operadores da álgebra relacional para validar a lógica de recuperação de informações antes de transformá-las em consultas definitivas.

Os dados estão organizados nas seguintes estruturas (esquemas):

  • VENDEDOR (Codigo_Vendedor, Nome_Vendedor, Salario_Fixo, Faixa_Comissao)
  • PEDIDO (Numero_Pedido, Valor_Total, Data_Pedido, Codigo_Vendedor)

Preparação

Para realizar esta atividade, você deve:

  • Revisar a simbologia grega: σ (Seleção), π (Projeção) e × (Produto Cartesiano).
  • Lembrar-se da propriedade do fechamento: o resultado de uma operação é sempre uma nova tabela que pode ser usada em uma operação seguinte.
  • Simular uma pequena massa de dados (3 vendedores e 5 pedidos) para validar os resultados visualmente.

Demonstração (Exemplo Básico)

Problema: Listar apenas os nomes dos vendedores que possuem salário fixo superior a 2.500,00.

Resolução Algébrica:

  1. Primeiro, filtramos as linhas (Seleção): T1 ← σ Salario_Fixo > 2500 (VENDEDOR)
  2. Depois, isolamos a coluna desejada (Projeção): π Nome_Vendedor (T1)

Expressão Única: π Nome_Vendedor (σ Salario_Fixo > 2500 (VENDEDOR))

O Desafio (Atividade)

Escreva as expressões de Álgebra Relacional para as seguintes solicitações de negócio:

  1. Filtro de Qualificação: Recupere todos os dados dos vendedores que pertencem à 'Faixa_Comissao' A.
  2. Relatório de Faturamento: Exiba apenas o número do pedido e o valor total de todos os pedidos realizados.
  3. Cruzamento Universal: Gere uma tabela que combine cada vendedor com todos os pedidos existentes (Produto Cartesiano).
  4. Consulta Combinada (Junção Lógica): Liste o nome do vendedor e o valor total de seus respectivos pedidos.
    Dica: Você precisará fazer o Produto Cartesiano e usar a Seleção para igualar as chaves: VENDEDOR.Codigo_Vendedor = PEDIDO.Codigo_Vendedor.
  5. Desafio de Otimização: Liste o nome dos vendedores que realizaram pedidos com 'Valor_Total' superior a 5.000,00. Aplique a Seleção o mais cedo possível para aumentar a performance.

Orientações Finais

  • Pense Proceduralmente: A álgebra relacional exige que você defina o "como" recuperar, passo a passo (de baixo para cima no fluxo lógico).
  • Atenção aos Parênteses: Assim como na matemática, as operações dentro dos parênteses internos são executadas primeiro.
  • Não confunda os Filtros: Lembre-se que a Seleção (σ) corta a tabela na horizontal (linhas) e a Projeção (π) corta na vertical (colunas).
  • Validação: Ao finalizar, tente traduzir mentalmente sua expressão para o SQL para verificar se a lógica faz sentido (SELECT = Projeção, FROM = Produto, WHERE = Seleção).
