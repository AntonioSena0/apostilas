Teoria & Conceitos

A manipulação de múltiplos dados relacionados é o que separa "scripts" de "sistemas". No Java, evoluímos de estruturas rígidas para coleções inteligentes.

1. Arrays (Vetores): A Base Rígida

Imagine um prédio com número fixo de apartamentos. Você não pode "puxar um puxadinho" depois de construído.

  • Contiguidade: Os dados ficam lado a lado na memória, o que torna o acesso extremamente rápido.
  • Tamanho Imutável: O length é definido no nascimento.
  • Tipos Primitivos: É a única estrutura que aceita int, double, etc., diretamente sem conversão.

Exemplo básico:

// Criando um array de 5 posições para números inteiros
int[] numeros = new int[5];
numeros[0] = 10;
numeros[1] = 20;
numeros[2] = 30;

// Acessando pelo índice
System.out.println("Elemento no índice 0: " + numeros[0]); // Saída: 10

// Iterando pelo array
for (int num : numeros) {
    System.out.println(num);
}

2. ArrayList: A Lista Elástica

O ArrayList é, na verdade, um Array que "se vira nos trinta". Quando ele enche, o Java cria um novo array maior nos bastidores, copia tudo e descarta o antigo.

  • Generics (<T>): O uso do diamante <String> evita que você misture "bananas com laranjas" na mesma lista.
  • Wrapper Classes: Como coleções só guardam Objetos, não podemos usar int. Usamos Integer. O Java faz a conversão automática (Autoboxing).

Exemplo básico:

// Criando um ArrayList que cresce automaticamente
List<String> frutas = new ArrayList<>();

// Adicionando elementos (sem definir tamanho prévio!)
frutas.add("Maçã");
frutas.add("Banana");
frutas.add("Laranja");

// Acessando um elemento
System.out.println(frutas.get(0)); // Saída: Maçã

// Obtendo o tamanho dinâmico
System.out.println("Total de frutas: " + frutas.size()); // Saída: 3

3. Performance: ArrayList vs LinkedList

Muitos alunos confundem os dois. Use esta regra de bolso:

  • ArrayList: É como uma fila de cadeiras. Para sentar alguém no meio, todo mundo depois dele tem que levantar e pular uma cadeira pro lado. (Lento para inserir no meio, rápido para achar alguém pelo número da cadeira).
  • LinkedList: É como uma corrente de pessoas de mãos dadas. Para colocar alguém no meio, basta soltar as mãos de dois vizinhos e dar a mão para o novo integrante. (Rápido para inserir, lento para achar alguém, pois tem que percorrer a corrente do início).

Exemplos comparativos:

// ArrayList - Excelente para acesso rápido
List<Integer> arrayList = new ArrayList<>();
arrayList.add(1);
arrayList.add(2);
arrayList.add(3);
int elemento = arrayList.get(0); // RÁPIDO - acesso direto

// LinkedList - Excelente para inserções no meio
List<Integer> linkedList = new LinkedList<>();
linkedList.add(1);
linkedList.add(2);
linkedList.add(3);
linkedList.add(1, 999); // RÁPIDO - insere no meio sem deslocar todos

4. Duelo de Gigantes: Qual Escolher?

Característica Array ([]) ArrayList (List)
Tamanho Fixo (Estático) Dinâmico (Cresce sozinho)
Performance Mais rápido (Menos memória) Levemente mais lento (Mais recursos)
Tipos Primitivos e Objetos Apenas Objetos (Wrapper Classes)
Facilidade Métodos manuais Métodos prontos (add, remove, sort)

5. Diferentes Implementações de Listas

A escolha da implementação impacta o desempenho dependendo da operação.

  • ArrayList: Rápida para acesso aleatório e iteração (busca pelo índice).
  • LinkedList: Muito eficiente para inserções e remoções no meio da lista ou nas extremidades.
  • Vector: Versão sincronizada (thread-safe) do ArrayList, porém mais lenta devido ao custo de bloqueios.

6. Estruturas Multi-Dimensionais

O Java permite criar tabelas de dados (linhas e colunas) organizando arrays dentro de arrays ou coleções aninhadas.

// --- MATRIZ (Array Bidimensional Fixo) ---
int[][] matriz = new int[3][3]; // 3 linhas e 3 colunas
matriz[0][0] = 10; // Linha 0, Coluna 0
matriz[1][2] = 25; // Linha 1, Coluna 2

// --- MATRIZ IRREGULAR (Cada linha com tamanho diferente) ---
int[][] irregular = { {1, 2}, {3, 4, 5}, {6} };

// --- LISTA MULTI-DIMENSIONAL (Dinâmica) ---
List<List<Integer>> lista2D = new ArrayList<>();
lista2D.add(new ArrayList<>(List.of(1, 2, 3))); // Linha 0
lista2D.add(new ArrayList<>(List.of(4, 5)));    // Linha 1

int valor = lista2D.get(0).get(2); // Acessa o valor 3

Exemplo de Código

Operações Completas do Dia a Dia

import java.util.ArrayList;
import java.util.List;
import java.util.Collections;

public class SupermercadoArrayList {
    public static void main(String[] args) {
        List<String> carrinho = new ArrayList<>();

        // --- ADIÇÃO ---
        carrinho.add("Arroz");
        carrinho.add("Feijão");
        carrinho.add(0, "Azeite"); // Insere no topo, empurra o resto

        // --- BUSCA E ACESSO ---
        if (!carrinho.isEmpty()) {
            System.out.println("Primeiro item: " + carrinho.get(0));
            System.out.println("Posição do Feijão: " + carrinho.indexOf("Feijão"));
        }

        // --- REMOÇÃO ---
        carrinho.remove("Arroz"); // Remove por valor
        carrinho.remove(0);       // Remove por índice

        // --- ORDENAÇÃO (O poder da classe Collections) ---
        Collections.sort(carrinho); 
        System.out.println("Carrinho Ordenado: " + carrinho);

        // --- ITERAÇÃO MODERNA (For-Each) ---
        for (String item : carrinho) {
            System.out.println("Item no checkout: " + item);
        }
    }
}

Dicas & Lembretes

  • length vs size(): Array usa a propriedade .length (sem parênteses). ArrayList usa o método .size() (com parênteses).
  • Índices: Ambos começam em 0. O último elemento está sempre em tamanho - 1.
  • A Pegadinha do remove: Ao remover um item pelo índice em um loop for tradicional, os índices dos itens restantes mudam! Sempre prefira o for-each ou o Iterator se precisar remover durante a leitura.

Como Aplicar na Prática

  • Criação de Vetor: Crie um array de 10 posições para armazenar nomes de produtos.
  • Leitura Dinâmica: Substitua o array por um ArrayList<String>. Use um loop while para permitir que o usuário insira nomes até digitar "sair".
  • Processamento: Use o método contains para verificar se o usuário está tentando adicionar um produto duplicado antes de usar o add.
  • Relatório: Exiba a lista ordenada alfabeticamente usando Collections.sort() e o laço for aprimorado for(String p : lista).
