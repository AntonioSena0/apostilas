Atividade Prática: Sistema MoneyBank - Comportamentos de Objetos

1. Enunciado

No paradigma de orientação a objetos, os métodos representam as ações ou comportamentos que um objeto pode realizar. Você deve evoluir a classe Conta do sistema MoneyBank, transformando-a de um simples repositório de dados em uma entidade funcional capaz de gerenciar saldo, validar operações financeiras e interagir com outras instâncias da mesma classe.

2. Preparação

  • Localização: Crie os arquivos dentro da pasta estruturada aulas/aula-12/exercicios/.
  • Arquivos Necessários:
    • MoneyBank.java: Classe principal (driver) que contém o método main para execução dos testes.
    • Conta.java: Classe que servirá de molde para os objetos, onde você implementará a lógica de negócio.
  • Requisitos: JDK 17+ instalado e ambiente de desenvolvimento configurado [Plano 370].

3. Código Base (Exemplo Básico)

O professor fornece a classe de teste pronta. O seu objetivo é fazer com que a classe Conta atenda a todas as chamadas realizadas abaixo.

MoneyBank.java 
public class MoneyBank {
    public static void main(String[] args) {
        Conta c1 = new Conta();
        Conta c2 = new Conta();

        c1.titular = "Aluno Herói";
        c1.saldo = 1000.0;
        
        c2.titular = "MoneyBank Reserva";
        c2.saldo = 500.0;

        System.out.println("--- Bem-vindo ao MoneyBank ---");
        
        // Testando Depósito
        System.out.println("\n[Ação] Depositando R$ 250 na conta de " + c1.titular);
        c1.deposita(250.0);
        System.out.println("Novo saldo c1: R$" + c1.getSaldo());

        // Testando Saque
        System.out.println("\n[Ação] Tentando sacar R$ 2000 da conta de " + c1.titular);
        if (c1.saca(2000.0)) {
            System.out.println("Saque realizado! Saldo atual: R$" + c1.getSaldo());
        } else {
            System.out.println("Erro: Saldo insuficiente.");
        }

        // Testando Transferência
        System.out.println("\n[Ação] Transferindo R$ 300 de " + c1.titular + " para " + c2.titular);
        if (c1.transfere(c2, 300.0)) {
            System.out.println("Transferência concluída com sucesso!");
        } else {
            System.out.println("Falha na transferência.");
        }

        System.out.println("\nSaldo final de " + c1.titular + ": R$" + c1.getSaldo());
        System.out.println("Saldo final de " + c2.titular + ": R$" + c2.getSaldo());
    }
}

4. O Desafio (Atividade em questão)

Sua tarefa é codificar a classe Conta.java implementando os seguintes métodos conforme a anatomia correta (cabeçalho e corpo):

  • Atributos: Declare os campos String titular e double saldo.
  • Método deposita (Sem Retorno):
    • Deve receber um double valor como parâmetro e somá-lo ao saldo atual.
    • Use a palavra-chave void para indicar que não há retorno de informação.
  • Método saca (Com Retorno Boolean):
    • Deve receber um double valor.
    • Lógica: Se o saldo for maior ou igual ao valor, subtraia e retorne true. Caso contrário, retorne false.
    • A instrução return encerra o método e devolve a resposta ao chamador.
  • Método getSaldo (Getter):
    • Deve retornar o valor contido no atributo saldo.
  • Método transfere (Interação entre Objetos):
    • Deve receber a Conta destino e o double valor.
    • Dica de Engenharia: Aplique a estratégia de "dividir para conquistar": use o método saca dentro da transferência para validar a operação antes de depositar na conta destino.

5. Orientações Finais

  • A Referência this: Utilize this.saldo para diferenciar variáveis de instância de parâmetros locais que possuam o mesmo nome.
  • Convenções: Nomes de métodos devem obrigatoriamente iniciar com letra minúscula (camelCase).
  • Modularização: Lembre-se que cada método deve realizar uma única tarefa bem definida para facilitar o reuso.
  • Validação: Verifique se o código compila sem erros. Se apagar um ponto e vírgula, a IDE indicará visualmente a falha antes mesmo da execução.
