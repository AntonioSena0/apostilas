Teoria & Conceitos

Esta aula foca nos Operadores Relacionais e Lógicos aplicados à linguagem Java, fundamentais para a construção de expressões booleanas que permitem ao programa tomar decisões e controlar o fluxo de execução.

1. Operadores Relacionais

Os operadores relacionais são utilizados para comparar dois valores do mesmo tipo. O resultado dessas comparações é sempre um valor lógico (boolean): true ou false. 

// Exemplo em Java
int idade = 18;
double media = 7.5;

System.out.println(idade >= 18); // Imprime: true
System.out.println(media < 6.0);  // Imprime: false
System.out.println(idade == 20); // Imprime: false
Operador Significado Exemplo em Java Resultado
== Igual a 5 == 5 true
!= Diferente de 5 != 3 true
> Maior que 10 > 20 false
< Menor que 10 < 20 true
>= Maior ou igual a 15 >= 15 true
<= Menor ou igual a 7 <= 5 false

Nota técnica: Em pseudocódigo, o símbolo de igualdade costuma ser apenas =, mas em Java utilizamos ==. O símbolo de "diferente" em Java é !=, substituindo o <> ou ≠ visto em algoritmos teóricos.

2. Operadores Lógicos

Os operadores lógicos, também chamados de conectivos lógicos, servem para unir duas ou mais expressões simples em uma expressão composta.

Operador Java Nome Descrição
&& E (Conjunção) Resulta em true apenas se ambas as expressões forem verdadeiras.
|| OU (Disjunção) Resulta em true se pelo menos uma das expressões for verdadeira.
! NÃO (Negação) Operador unário que inverte o valor lógico atual. 
boolean temIngresso = true;
boolean temIdadeMinima = false;

// Operador E (&&)
System.out.println(temIngresso && temIdadeMinima); // false (precisa de ambos)

// Operador OU (||)
System.out.println(temIngresso || temIdadeMinima); // true (um deles é verdadeiro)

// Operador NÃO (!)
System.out.println(!temIngresso); // false (inverteu true para false)

4. Precedência de Operadores

Quando uma expressão possui diversos tipos de operadores, o Java segue uma ordem de prioridade para a avaliação:

  • Parênteses (): Sempre avaliados primeiro, dos mais internos para os externos.
  • Operadores Aritméticos: Multiplicação, divisão e resto (*, /, %) antes de soma e subtração.
  • Operadores Relacionais: >, <, >=, <=, ==, !=.
  • Operadores Lógicos: A ordem interna é ! (Negação), seguido de && (E) e, por último, || (OU). 
int x = 10, y = 5, z = 20;
// Avaliação: (10 > 5) && (20 < 100) -> (true) && (true) -> true
boolean resultado = x > y && z < 100;

// Avaliação: (5 + 5 == 10) || (false) -> (10 == 10) || false -> true || false -> true
boolean teste = (y + 5 == x) || !(z > 10);

Resumo

  • Relacionais comparam valores (números, caracteres).
  • Lógicos comparam condições (booleanos).
  • Use parênteses para garantir que a lógica complexa seja avaliada na ordem que você deseja, aumentando a clareza do código.

Exemplo de Código

Exemplo 1: Operadores Relacionais Básicos

Este exemplo demonstra como comparar valores numéricos simples.

public class RelacionaisBasico {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        System.out.println("a == b: " + (a == b)); // Igualdade (false)
        System.out.println("a != b: " + (a != b)); // Diferença (true)
        System.out.println("a > b: " + (a > b));   // Maior que (false)
        System.out.println("a < b: " + (a < b));   // Menor que (true)
        System.out.println("a >= 10: " + (a >= 10)); // Maior ou igual (true)
    }
}

Exemplo 2: Operador Lógico E (&&)

Utilizado quando todas as condições precisam ser verdadeiras. Baseado no conceito de conjunção.

public class VerificacaoAcesso {
    public static void main(String[] args) {
        int idade = 25;
        boolean possuiConvite = true;

        // Para entrar na festa, precisa ter idade >= 18 E possuir convite
        boolean podeEntrar = (idade >= 18) && possuiConvite;

        System.out.println("Pode entrar na festa? " + podeEntrar); // true
    }
}

Exemplo 3: Operador Lógico OU (||)

Utilizado quando pelo menos uma das condições deve ser verdadeira. Baseado no conceito de disjunção.

public class DescontoEstudante {
    public static void main(String[] args) {
        boolean eEstudante = false;
        boolean temCupom = true;

        // Tem desconto se for estudante OU se tiver cupom
        boolean temDireitoADesconto = eEstudante || temCupom;

        System.out.println("Tem desconto? " + temDireitoADesconto); // true
    }
}

Exemplo 4: Operador Lógico NÃO (!)

Utilizado para inverter um valor booleano. Baseado no conceito de negação.

public class StatusSistema {
    public static void main(String[] args) {
        boolean sistemaOffline = false;

        // Se NÃO estiver offline, o sistema está online
        if (!sistemaOffline) {
            System.out.println("O sistema está ONLINE e operando.");
        }
    }
}

Exemplo 5: Cenário Prático (Aprovação Escolar)

Este exemplo combina múltiplos operadores para verificar a situação de um aluno, conforme sugerido nos fundamentos de lógica.

public class SituacaoAluno {
    public static void main(String[] args) {
        double mediaFinal = 6.5;
        double frequencia = 0.80; // 80% de presença

        // Critério: Média >= 7.0 E Frequência >= 0.75
        boolean aprovado = (mediaFinal >= 7.0) && (frequencia >= 0.75);

        // Critério para Recuperação: Frequência ok, mas média entre 5.0 e 6.9
        boolean recuperacao = (frequencia >= 0.75) && (mediaFinal >= 5.0 && mediaFinal < 7.0);

        System.out.println("Aprovado: " + aprovado);       // false
        System.out.println("Recuperação: " + recuperacao); // true
    }
}

Exemplo 6: Precedência e Parênteses

Demonstra como a ordem de avaliação afeta o resultado final.

public class TestePrecedencia {
    public static void main(String[] args) {
        boolean p = true;
        boolean q = false;
        boolean r = false;

        // Sem parênteses: ! tem prioridade, depois &&, depois ||
        // Avaliação: true || (false && false) -> true || false -> true
        boolean resultado1 = p || q && r;

        // Com parênteses alterando a ordem:
        // Avaliação: (true || false) && false -> true && false -> false
        boolean resultado2 = (p || q) && r;

        System.out.println("Resultado 1: " + resultado1);
        System.out.println("Resultado 2: " + resultado2);
    }
}

Dicas & Lembretes

  • Atribuição vs. Comparação: Nunca confunda o operador de atribuição (=), usado para guardar um valor em uma variável, com o operador de igualdade (==), usado para comparar se dois valores são iguais.
  • Resultado Booleano: Lembre-se sempre que qualquer expressão que utilize operadores relacionais ou lógicos resultará obrigatoriamente em um valor do tipo lógico (boolean): true ou false.
  • Hierarquia de Execução: Na falta de parênteses, o Java segue uma ordem de prioridade rigorosa: primeiro resolve a negação (!), depois a conjunção (&&) e, por fim, a disjunção (||).
  • O Poder dos Parênteses: Utilize parênteses () não apenas para alterar a ordem de precedência, mas também para tornar o código mais legível para outros programadores.
  • Proibição de Comparações em Cadeia: Diferente da matemática, você não pode escrever 10 < x < 20 diretamente no código; em Java, você deve separar as condições usando um operador lógico, como (x > 10) && (x < 20).
  • Curto-Circuito Lógico: O Java utiliza a avaliação de curto-circuito; em uma expressão &&, se a primeira parte for falsa, a segunda nem é avaliada, pois o resultado final será obrigatoriamente falso.
  • Negação Inversora: O operador unário ! tem a função simples de inverter o estado lógico; o que é verdadeiro torna-se falso e vice-versa.
  • Diferença entre OU e XOR: Enquanto o operador OU (||) aceita que ambas as condições sejam verdadeiras, o OU Exclusivo (^) exige que os operandos sejam diferentes entre si para retornar true.
  • Tipagem de Dados: Ao realizar comparações relacionais, certifique-se de que os dados são de tipos compatíveis para evitar erros de compilação.
  • Prática com Tabelas-Verdade: Em caso de dúvida sobre lógica complexa, rascunhe uma Tabela-Verdade para validar todas as combinações possíveis de entrada e garantir que seu algoritmo tome a decisão correta.

Como aplicar na Prática

  • Sistemas de Autenticação e Segurança: Cenário: Validar se um usuário pode acessar uma área administrativa.
    Lógica: (usuarioValido == true) && (senhaCorreta == true) && (perfilAdmin == true).
    Aplicação: Uso do operador && para garantir que múltiplos critérios de segurança sejam atendidos simultaneamente.
  • E-commerce e Promoções: Cenário: Aplicar desconto em uma compra.
    Lógica: (valorCompra > 500.0) || (cupomAtivo == true).
    Aplicação: Uso do operador ||, onde o desconto é concedido se o cliente gastar muito ou se possuir um código promocional.
  • Controle de Acesso por Idade (Streaming/Jogos): Cenário: Verificar se um conteúdo é adequado para o espectador.
    Lógica: (idadeUsuario >= classificacaoIndicativa) || (autorizacaoPais == true).
    Aplicação: Operadores relacionais de maior ou igual combinados com a lógica de exceção.
  • Sistemas Acadêmicos (Aprovação): Cenário: Decidir se um aluno foi aprovado, reprovado ou está em recuperação.
    Lógica: (mediaFinal >= 7.0) && (frequencia >= 0.75).
    Aplicação: Conjunção lógica para verificar se o aluno atingiu o desempenho acadêmico e a presença mínima necessária.
  • Fintechs e Análise de Crédito: Cenário: Aprovar um empréstimo bancário.
    Lógica: (salario > limiteMinimo) && !(possuiRestricaoNome).
    Aplicação: Uso do operador de negação ! para verificar se o cliente não possui restrições financeiras.
  • Desenvolvimento de Jogos (Colisão e Limites): Cenário: Verificar se um personagem está dentro dos limites da tela.
    Lógica: (posX >= 0) && (posX <= larguraTela).
    Aplicação: Delimitação de intervalos numéricos que o computador deve monitorar constantemente para atualizar a interface gráfica.
  • Sistemas de Logística e Estoque: Cenário: Alertar sobre a necessidade de reposição de produtos.
    Lógica: (quantidadeEstoque < estoqueMinimo) && (fornecedorDisponivel == true).
    Aplicação: Comparação relacional de menor que para automação de pedidos de compra.

Resumo prático: Em projetos reais, você raramente usará um operador isolado. A "mágica" da programação acontece quando você encadeia essas comparações para modelar situações complexas do mundo real dentro do código Java.
