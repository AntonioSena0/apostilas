Enunciado

Você está desenvolvendo o motor lógico de um jogo de RPG em turnos. Sua tarefa é programar a mecânica de "Dano e Esquiva". O programa deve decidir se um ataque atingiu o oponente, se houve um acerto crítico e qual a mensagem de status do defensor baseada em sua vida restante.

Preparação

  • Certifique-se de ter o JDK instalado e um editor de código.
  • Crie uma classe chamada SimuladorCombate.
  • Utilizaremos a classe Scanner para entrada de dados e a classe Random para simular a sorte (dados).

Código Base

import java.util.Scanner;
import java.util.Random;

public class SimuladorCombate {
    public static void main(String[] args) {
        Scanner teclado = new Scanner(System.in);
        Random gerador = new Random();

        System.out.print("Pontos de Vida (HP) do Inimigo: ");
        int hpInimigo = teclado.nextInt();

        // Sorteia um valor de ataque entre 1 e 20 (como um dado D20)
        int valorAtaque = gerador.nextInt(20) + 1;
        System.out.println("Você rolou um: " + valorAtaque);

        // --- O DESAFIO COMEÇA AQUI ---
        
        int danoBase = 10;
        
        // Implemente a lógica de acerto, dano e status...

        // --- O DESAFIO TERMINA AQUI ---
        
        teclado.close();
    }
}

O Desafio (Atividade)

Implemente as seguintes regras utilizando estruturas de condição:

  • Condição Simples (Esquiva): Se o valorAtaque for menor que 5, o ataque falhou. Imprima "O inimigo esquivou!" e encerre a execução usando return;.
  • Condição Composta (Acerto Crítico):
    • Se o valorAtaque for igual a 20, o dano é dobrado (danoBase * 2) e a mensagem "ACERTO CRÍTICO!" deve ser exibida.
    • Caso contrário (ataque normal entre 5 e 19), o dano causado é apenas o danoBase.
  • Estrutura Encadeada (Status do Inimigo): Após calcular o dano e subtrair do hpInimigo, verifique o estado atual dele:
    • HP menor ou igual a 0: "Inimigo Derrotado!"
    • HP entre 1 e 10: "Inimigo Ferido Gravemente!"
    • HP maior que 10: "Inimigo ainda está de pé!"

Orientações Finais

  • Uso de Chaves: Mantenha a boa prática de utilizar chaves { } em todos os blocos if e else, facilitando a leitura e futuras expansões do código do jogo.
  • Identação: Organize o código para que fique claro quais ações dependem de quais condições, especialmente na parte do cálculo de dano.
  • Aplicação Prática: Note como a lógica de programação é o que define a "inteligência" básica e as regras de um jogo, permitindo diferentes resultados para cada jogada.
  • Dica para os alunos: No RPG, as condições são o que chamamos de "mecânicas de jogo". Sem o if/else, o jogo sempre teria o mesmo resultado!
