Enunciado

Nesta atividade, você deve construir um script de processamento para um formulário de cadastro de eventos. O objetivo é garantir que nenhum dado malicioso (scripts XSS) entre no sistema e que todas as regras de negócio (idade mínima, formato de e-mail e URL) sejam respeitadas utilizando estritamente as funções de filtro nativas do PHP.

Preparação

  • Ambiente: Certifique-se de usar o PHP 8.1 ou superior.
  • Arquivos: Crie uma pasta e salve os dois arquivos abaixo (index.html e processa.php).
  • Tipagem Estrita: O arquivo PHP deve conter declare(strict_types=1); no topo para evitar que o PHP converta tipos de forma automática e insegura.

Orientações Finais e Testes

  • Regra de Ouro: Não acesse $_POST diretamente. Use apenas filter_input().
  • Tratamento de Erros: Caso algum campo seja inválido, o programa deve listar os erros em vez de confirmar o cadastro.
  • Teste de Segurança: No campo "Nome", tente enviar: <script>alert('hack')</script>. O PHP deve exibir o texto na tela em vez de executar o alerta, provando que a sanitização funcionou.
  • Dica Profissional: O parâmetro options do filter_input permite definir min_range e max_range, o que economiza vários blocos de if no seu código.
