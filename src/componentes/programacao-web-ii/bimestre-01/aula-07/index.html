Teoria & Conceitos

A segurança em PHP baseia-se em uma regra de ouro: Nunca confie nos dados que vêm do usuário. Dados externos (formulários, URLs, cookies) são "sujos" por natureza e podem conter desde erros de digitação até códigos maliciosos.

1. Validação vs. Sanitização

Embora pareçam iguais, elas possuem objetivos distintos e complementares:

  • Sanitização (A Limpeza): É o processo de remover ou transformar caracteres indesejados. O PHP faz uma "faxina" no dado para torná-lo seguro para exibição ou armazenamento.
    • Cenário: O usuário envia <h1>Olá</h1> em um campo de nome.
    • Ação: A sanitização remove as tags, restando apenas Olá.
  • Validação (A Verificação): É o processo de checar se o dado segue uma regra específica (tipo, tamanho, formato). O PHP pergunta: "Este dado é aceitável?".
    • Cenário: O sistema espera uma idade (número). O usuário digita "vinte".
    • Ação: A validação falha e o sistema recusa o dado.

2. O perigo da Manipulação de Tipos (Type Juggling)

O PHP é uma linguagem fracamente tipada, o que pode ser uma armadilha de segurança. Ele tenta "adivinhar" o que você quer fazer, mas um atacante pode usar isso a favor dele.

O Problema: Se você espera uma string e o PHP recebe um Array ou um Boolean, funções que manipulam strings podem gerar erros fatais (Fatal Errors) e derrubar o site.

Exemplo Prático do Perigo:

// Código inseguro
$nome = $_POST['nome']; 
echo strtoupper($nome); // Se o atacante enviar 'nome[]' via formulário, $nome vira um array e o PHP trava.

A solução é validar não apenas o conteúdo, mas garantir que o dado seja do tipo esperado (ex: garantir que ID seja sempre um int).

3. A solução moderna: filter_input()

Acessar $_POST ou $_GET diretamente é considerado uma má prática hoje em dia. O filter_input() é a ferramenta oficial do PHP para lidar com entradas externas de forma segura.

  • Vantagem 1 (Existência): Ele não gera erro se o campo não foi enviado (retorna NULL).
  • Vantagem 2 (Filtro Nativo): Ele permite sanitizar ou validar o dado no exato momento em que ele "entra" no seu script.

Comparação rápida:

  • Jeito Antigo: $email = $_POST['email']; (Inseguro e gera aviso se o campo estiver vazio).
  • Jeito Moderno: $email = filter_input(INPUT_POST, 'email', FILTER_SANITIZE_EMAIL); (Seguro, limpo e profissional).

Exemplo de Código

<?php
// 1. Sanitização de Nome (Remove tags HTML)
$nome = filter_input(INPUT_POST, 'nome', FILTER_SANITIZE_SPECIAL_CHARS);

// 2. Validação de Idade (Garante que é um número entre 18 e 100)
$idade = filter_input(INPUT_POST, 'idade', FILTER_VALIDATE_INT, [
    "options" => ["min_range" => 18, "max_range" => 100]
]);

// 3. Validação de E-mail
$email = filter_input(INPUT_POST, 'email', FILTER_VALIDATE_EMAIL);

if ($email === false) {
    echo "E-mail inválido!";
} else {
    echo "E-mail válido: $email";
}
?>

Dicas & Lembretes

  • Sempre use filter_input() em vez de acessar globais como $_POST diretamente.
  • Sanitização limpa o que entra; Validação decide se pode entrar.

Como aplicar na Prática

Ao criar formulários de cadastro ou login, utilize o filter_input para cada campo. Isso previne ataques de XSS (Cross-Site Scripting) e garante que o banco de dados não receba lixo eletrônico.
