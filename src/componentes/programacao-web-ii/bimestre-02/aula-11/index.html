1. Teoria & Conceitos

A Programação Orientada a Objetos (POO) organiza o software em torno de "objetos" em vez de apenas funções e lógica sequencial. Ela reconhece que dados (propriedades) e comportamentos (métodos) devem estar unidos em uma única estrutura.

Conceitos com Exemplos Práticos:

  • Classe (O Molde): É um modelo ou plano para criar algo.
    Exemplo Prático: Pense na planta de arquitetura de uma casa. A planta define onde ficam as janelas e portas, mas você não pode morar na planta. Ela é apenas a definição teórica do que uma casa deve ter. 
    // A classe define o molde (planta)
class Casa {
    public $cor;
    public $voltagem;
}
  • Objeto (A Instância): É a concretização da classe.
    Exemplo Prático: Utilizando a planta (Classe), você constrói uma casa real na Rua A e outra na Rua B. Ambas seguem o mesmo molde, mas são casas distintas (objetos), cada uma com sua própria cor ou morador. 
    // Criando objetos reais a partir do molde
$casaRuaA = new Casa();
$casaRuaB = new Casa();

$casaRuaA->cor = "Azul";
$casaRuaB->cor = "Branca";
  • Propriedades (Atributos): São as características ou dados que o objeto armazena.
    Exemplo Prático: Em um sistema de biblioteca, um objeto Livro teria propriedades como $titulo, $autor e $numeroPaginas. 
    class Livro {
    public $titulo;
    public $autor;
    public $numeroPaginas;
}
  • Métodos (Comportamentos): São as ações que o objeto pode realizar.
    Exemplo Prático: O objeto Livro pode ter um método abrir() ou trocarPagina(). São as funções que operam sobre os dados internos daquele objeto específico. 
    class Livro {
    public function abrir() {
        echo "O livro agora está aberto.";
    }
}
  • Visibilidade (Encapsulamento): Define quem pode ver ou alterar os dados.
    • Public: Acessível de qualquer lugar (dentro ou fora da classe).
    • Protected: Acessível apenas dentro da própria classe e por classes que herdam dela (filhas).
    • Private: Acessível apenas dentro da própria classe (protege os dados contra alterações externas indevidas).
    class Conta {
    public $titular;    // Aberto
    protected $token;   // Herança
    private $saldo;     // Proteção total
}

2. Exemplo de Código

Abaixo, apresento a implementação de uma classe para gerenciar contas bancárias e o uso de Promoção de Propriedades do PHP 8.

<?php
/**
 * Cenário Básico: Definição e Instanciação
 */
class ContaBancaria {
    public string $titular; // Propriedade pública
    private float $saldo = 0.0; // Propriedade privada: protegida contra acesso direto

    // Método para depositar: ação controlada sobre o saldo
    public function depositar(float $valor): void {
        $this->saldo += $valor; // $this refere-se à instância atual
        echo "Depósito de R$ {$valor} realizado para {$this->titular}.\n";
    }

    // Método para consultar o saldo: interface segura para ler dado privado
    public function consultarSaldo(): float {
        return $this->saldo;
    }
}

// Instanciação: Criando um objeto real
$minhaConta = new ContaBancaria();
$minhaConta->titular = "Roni Sommerfeld";
$minhaConta->depositar(500.00);

echo "Saldo atual: R$ " . $minhaConta->consultarSaldo();

/**
 * Recurso Avançado: Promoção de Propriedades (PHP 8)
 */
class Usuario {
    // O construtor é chamado automaticamente no 'new'
    public function __construct(
        public string $nome,  // Declara e atribui simultaneamente
        public string $email  
    ) {
        // A atribuição ocorre de forma implícita e automática pelo PHP 8
        echo "Usuário {$this->nome} criado com sucesso!\n"; 
    }
}

$user = new Usuario("Ana Silva", "ana@email.com");
?>

4. Dicas & Lembretes

  • Identidade Própria: Mesmo que dois objetos venham da mesma classe, eles são independentes na memória; alterar um não afeta o outro.
  • Encapsulamento é Lei: Prefira sempre propriedades private ou protected com métodos de acesso para validar os dados.
  • Case Sensitivity: Nomes de classes são case-insensitive, mas as variáveis de objeto são case-sensitive ($conta != $Conta).

5. Como aplicar na Prática

Em um sistema de e-commerce, em vez de lidar com arrays de strings para cada peça, você cria uma classe Peca. Veja a diferença de valor que a POO entrega:

Abordagem Procedural (Sem POO)

Os dados ficam soltos e a lógica de negócio (como descontos) precisa ser repetida manualmente em todo o sistema.

<?php
$peca_nome = "Pistão";
$peca_preco = 500.00;
$desconto = $peca_preco * 0.10; // Lógica exposta e repetitiva
$final = $peca_preco - $desconto;

Abordagem Orientada a Objetos (Com POO)

A inteligência está centralizada. Se a regra de desconto mudar, você altera apenas um lugar.

<?php

class Peca {
  public function __construct(public string $nome, private float $preco) {}

  public function calcularPrecoComDesconto(float $percentual): float {
      // Regra de negócio protegida e centralizada
      return $this->preco - ($this->preco * ($percentual / 100));
  }
}

$p1 = new Peca("Pistão", 500.00);
echo $p1->calcularPrecoComDesconto(10);
