Teoria & Conceitos

1. O que é CRUD?

CRUD é um acrônimo para as quatro operações fundamentais que um sistema realiza em um banco de dados: Create (Criar), Read (Ler), Update (Atualizar) e Delete (Excluir). Em uma aplicação Web moderna, essas operações são mapeadas para os verbos do protocolo HTTP (POST, GET, PUT/PATCH e DELETE).

No Laravel, o CRUD deixa de ser uma série de scripts isolados e passa a ser gerenciado de forma centralizada através do Eloquent ORM e dos Resource Controllers.

2. A Estrutura do CRUD no Laravel

O desenvolvimento de um CRUD no framework é distribuído entre as camadas da arquitetura MVC (Model-View-Controller):

  • Model (Modelo): Representa a tabela no banco de dados. Utiliza o Eloquent para realizar as operações sem a necessidade de escrever SQL puro. Por exemplo, o comando Produto::all() substitui um SELECT * FROM produtos.
  • Controller (Controlador): Atua como o "policial de trânsito", recebendo a requisição do usuário, validando os dados, solicitando que a Model salve a informação e, por fim, retornando uma resposta (view ou redirecionamento).
  • View (Visão): Utiliza o motor de templates Blade para renderizar os formulários de cadastro e as tabelas de listagem de dados.

3. Resource Controllers: Padronização de Mercado

Um dos grandes diferenciais do Laravel é a convenção de nomes para as ações de um CRUD. Em vez de inventar nomes de funções, utilizamos os Resource Controllers, que criam automaticamente sete métodos padrão para lidar com o recurso:

Verbo HTTP URI Método do Controller Ação do CRUD
GET /produtos index Listar todos os registros.
GET /produtos/create create Exibir o formulário de novo cadastro.
POST /produtos store Salvar os dados no banco de dados.
GET /produtos/{id} show Exibir um registro específico.
GET /produtos/{id}/edit edit Exibir o formulário de edição.
PUT/PATCH /produtos/{id} update Atualizar os dados no banco de dados.
DELETE /produtos/{id} destroy Remover o registro do banco.

Para gerar essa estrutura completa de uma só vez, utilizamos o comando Artisan: php artisan make:controller ProdutoController --resource.

4. Persistência e Segurança Automática

Ao realizar um CRUD com Laravel, o desenvolvedor ganha camadas de proteção integradas:

  • Proteção CSRF: O Laravel exige um token de segurança em todos os formulários POST/PUT/DELETE para evitar ataques de falsificação de solicitação entre sites. No Blade, basta usar a diretiva @csrf.
  • Mass Assignment (Atribuição em Massa): Para evitar que usuários malintencionados enviem dados extras que não deveriam ser alterados (como um campo is_admin), o Laravel exige que você defina quais campos são "preenchíveis" através da propriedade $fillable na sua Model.
  • Prepared Statements: O Eloquent utiliza internamente o PDO com consultas parametrizadas, o que neutraliza automaticamente ataques de SQL Injection.

5. Exemplo de Fluxo: Criando um Registro (Store)

No Laravel, o processo de criação segue uma lógica fluida:

  1. Captura: O Controller recebe um objeto Request contendo todos os dados do formulário.
  2. Validação: Os dados são validados (ex: required, min:3) antes de tocar o banco.
  3. Persistência: A Model usa o método create() para inserir o registro.
  4. Redirecionamento: O usuário é enviado de volta com uma mensagem de sucesso, evitando o reenvio de dados ao atualizar a página (Padrão PRG).

Onde os arquivos ficam localizados?

  • Rotas: routes/web.php
  • Controllers: app/Http/Controllers/
  • Models: app/Models/
  • Views: resources/views/
