Enunciado

O objetivo desta atividade é simular o backend de um sistema de gestão de cursos. Você deverá configurar e manipular três tipos de relações:

  • Um-para-Um: Um Aluno (Student) possui um Cartão de Estudante (StudentCard).
  • Um-para-Muitos: Um Professor (Teacher) ministra muitos Cursos (Course).
  • Muitos-para-Muitos: Alunos (Student) inscrevem-se em muitos Cursos (Course), com uma coluna extra de "nota" (grade) na tabela intermediária.

Preparação

Antes de abrir o terminal, certifique-se de que as suas Migrations e Models estão configuradas corretamente:

  • Migrations: Devem conter as chaves estrangeiras (foreignId).
  • Models: Devem conter os métodos hasOne, belongsTo, hasMany e belongsToMany (este último com withPivot('grade')).
  • Factories: Devem estar prontas para gerar dados fictícios.

1. Código Base: Tinker (Exemplos de Referência)

Para realizar os desafios, você utilizará os seguintes comandos padrão no Tinker:

// Abrir o Tinker
php artisan tinker

// Exemplo 1: Criar com relacionamento automático (Factory)
$student = Student::factory()->has(StudentCard::factory())->create();

// Exemplo 2: Associar Um-para-Muitos
$teacher = Teacher::factory()->create();
$course = Course::factory()->for($teacher)->create();

// Exemplo 3: Sincronizar Muitos-para-Muitos com dados Pivot
$student->courses()->sync([
    $courseId => ['grade' => 9.5]
]);

O Desafio (Atividades)

Realize as seguintes operações dentro do terminal php artisan tinker:

Tarefa A: Identidade Estudantil (1:1)

  • Crie um Student usando a factory.
  • Crie um StudentCard e associe-o manualmente ao aluno criado.
  • Teste: Tente acessar o código do cartão a partir do objeto aluno: $student->studentCard->code.

Tarefa B: Grade Escolar (1:N)

  • Crie um Teacher chamado "Mestre Laravel".
  • Utilize a factory para gerar 3 Course vinculados a este professor de uma só vez.
  • Teste: Liste apenas os nomes dos cursos desse professor usando $teacher->courses->pluck('name').

Tarefa C: Inscrições e Notas (N:N)

  • Crie 2 alunos e 2 cursos.
  • Inscreva o Aluno 1 nos dois cursos, atribuindo notas diferentes via sync.
  • Inscreva o Aluno 2 apenas no primeiro curso.
  • Teste: Recupere a nota do Aluno 1 no curso específico usando $student->courses->first()->pivot->grade.

Orientações Finais

  • Persistência: Lembre-se que comandos no Tinker gravam dados reais no seu banco de dados. Use DatabaseTransactions se estiver num ambiente de testes.
  • Erros Comuns: Se receber um erro de "Class not found", verifique os namespaces no topo dos seus arquivos Models.
  • Refresh: Se alterar o código PHP de um Model enquanto o Tinker estiver aberto, você deve sair (exit) e entrar novamente para que as mudanças façam efeito.
  • Dica: Use o comando vanderloost/tinker-companion ou simplesmente o método dd() (dump and die) para inspecionar coleções complexas se necessário.
