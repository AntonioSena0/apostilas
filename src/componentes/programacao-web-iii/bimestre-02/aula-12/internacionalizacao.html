Configurando o Idioma

Para deixar as mensagens de erro do Laravel em português (PT-BR), você precisa configurar a localização do framework e adicionar os arquivos de tradução correspondentes. Como o Laravel não vem com as traduções de todos os idiomas por padrão, seguimos estes passos:

1. Alterar o Idioma no Arquivo de Configuração

Abra o arquivo config/app.php e altere a chave locale de en para pt_BR.

'locale' => 'pt_BR',
'fallback_locale' => 'en',

2. Adicionar os Arquivos de Tradução

Você tem duas formas principais de obter as traduções:

Opção A: Manualmente (Pasta lang)

No Laravel 10, 11, 12 e 13, os arquivos de tradução ficam na pasta raiz /lang.

  • Crie uma pasta chamada pt_BR dentro de /lang.
  • Adicione os arquivos: auth.php, pagination.php, passwords.php e validation.php.
  • Você pode baixar esses arquivos prontos do repositório da comunidade Laravel-Lang.

Opção B: Via Composer (Recomendado)

A forma mais rápida e profissional é usar o pacote da comunidade que gerencia centenas de idiomas automaticamente.

  • Instale o publicador: 
    composer require laravel-lang/common --dev
  • Adicione o idioma PT-BR: 
    php artisan lang:add pt_BR

Este comando criará automaticamente a pasta /lang/pt_BR com todas as validações (como "O campo :attribute é obrigatório") já traduzidas.

3. Traduzir Nomes de Campos (Attributes)

Para que o erro não apareça como "O campo email é obrigatório", você deve definir apelidos amigáveis no arquivo lang/pt_BR/validation.php, dentro do array attributes:

'attributes' => [
    'email' => 'endereço de e-mail',
    'password' => 'senha',
    'name' => 'nome completo',
],

Dicas & Lembretes

  • Limpeza de Cache: Após alterar o config/app.php, é recomendável limpar o cache de configuração para garantir que o Laravel reconheça a mudança: 
    php artisan config:clear
  • JSON de Tradução: Se você usa tradução via strings (ex: __('Welcome')), você pode criar um arquivo /lang/pt_BR.json para mapear essas frases.
  • Fallback: O fallback_locale em config/app.php serve como "plano B". Se o Laravel não encontrar uma tradução em português, ele usará a definida ali (geralmente en).

Como aplicar na Prática

Imagine que você tem um formulário de cadastro. Com essa configuração, ao falhar na validação, o Laravel retornará automaticamente:

  • Antes: "The email field is required."
  • Depois: "O campo endereço de e-mail é obrigatório."
